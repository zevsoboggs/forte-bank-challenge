# СЛАЙД 1: ПРОБЛЕМА

## Анализ тендеров — это боль

```
📊 Текущая ситуация:
├── Данные на 10+ разных площадках
├── Разные форматы документов (PDF, Excel, HTML)
├── Ручной сбор занимает часы
├── Риски: аффилированность, завышенные цены
└── Аналитик обрабатывает 5-10 тендеров в день
```

### Боли закупщика:
- **Время** - часы на сбор информации с разных площадок
- **Пропуски** - не успевают отследить все релевантные тендеры
- **Риски** - сложно выявить аффилированных поставщиков
- **Решения** - нет данных для обоснованного выбора

### Алгоритм скоринга:
```python
risk_score = 0

# Ценовая аномалия
if price > market_avg * 1.3:
    risk_score += 25

# Аффилированность
if check_affiliation(customer, suppliers):
    risk_score += 35

# Подозрительные сроки
if deadline_days < 5:
    risk_score += 20

# История поставщика
if supplier_complaints > 3:
    risk_score += 20
```
