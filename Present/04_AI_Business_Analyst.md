# СЛАЙД 1: ПРОБЛЕМА

## Бизнес-аналитики перегружены рутиной

```
📊 Текущая ситуация:
├── 3-5 дней на сбор требований по одной задаче
├── Бесконечные уточняющие встречи
├── Документация устаревает быстрее чем пишется
├── Use Cases пишутся вручную
└── Confluence заполняется неделями
```

### Боли бизнес-аналитика:
- **Время** - дни на формализацию одного требования
- **Качество** - упущенные детали из-за спешки
- **Коммуникация** - постоянные уточнения у заказчика
- **Документация** - ручное создание артефактов

> **Факт:** 60% времени BA тратит на документирование, а не на анализ

2. Переводы >100,000 ₸ - дополнительная верификация

## KPI
- Среднее время: ≤5 сек (цель)

# СЛАЙД 6: ГЕНЕРАЦИЯ АРТЕФАКТОВ (7 баллов)

## Use Cases, User Stories, диаграммы

### Use Case (автогенерация):
```
┌─────────────────────────────────────────┐
│  UC-001: Быстрый перевод               │
├─────────────────────────────────────────┤
│  Actor: Клиент                          │
│  Precondition: Авторизован в приложении │
│                                         │
│  Main Flow:                             │
│  1. Клиент выбирает "Перевод"          │
│  2. Вводит сумму и получателя          │
│  3. Система проверяет лимит            │
│  4. Если ≤100К - мгновенное подтв.     │
│  5. Показать статус "Выполнено"        │
│                                         │
│  Alternative Flow:                      │
│  4a. Если >100К - запрос SMS-кода      │
│                                         │
│  Postcondition: Перевод выполнен       │
└─────────────────────────────────────────┘
```

### User Story (автогенерация):
```
Как клиент мобильного банка,
Я хочу мгновенно подтверждать переводы до 100,000 ₸,
Чтобы не тратить время на ожидание.

Acceptance Criteria:
✅ Время подтверждения ≤5 сек
✅ Работает для переводов ≤100К
✅ Показывает статус в реальном времени
```

### Диаграмма процесса:
```
[Клиент] → [Ввод данных] → [Проверка лимита]
                               ↓
              ┌────────────────┴────────────────┐
              ↓                                 ↓
        [≤100K: Instant]              [>100K: SMS verify]
              ↓                                 ↓
              └────────────────┬────────────────┘
                               ↓
                         [Выполнено]
```

# СЛАЙД 13: ПРИМЕРЫ ГЕНЕРАЦИИ

## Реальные выходные документы

### BRD (фрагмент):
```markdown
# Бизнес-требование: Мгновенное открытие счёта

## 1. Цель
Сократить время открытия счёта с 2 дней до 15 минут
для физических лиц с подтверждённой идентификацией.

## 2. Бизнес-контекст
Текущий процесс требует ручной верификации документов,
что создаёт bottleneck в отделе операций.

## 3. Scope
### В scope:
- Открытие текущего счёта
- Открытие депозита
- Физические лица - резиденты РК

### Вне scope:
- Юридические лица
- Нерезиденты
- Специальные счета

## 4. KPI
| Метрика | Текущее | Цель |
|---------|---------|------|
| Время открытия | 48 часов | 15 минут |
| Конверсия заявок | 60% | 85% |
| NPS | 45 | 70 |
```
