# СЛАЙД 1: ПРОБЛЕМА

## Scrum-процессы съедают время команды

```
📊 Боли проектных команд:
├── 2-3 часа/неделю на создание задач в Jira
├── 30% времени stand-up уходит на статусы
├── Решения с встреч теряются
├── Дедлайны пропускаются
└── PM/Scrum Master перегружен рутиной
```

### Типичный день Scrum Master:
```
09:00 - Stand-up (15 мин говорим, 30 мин записываем)
10:00 - Создание задач из вчерашних решений
11:00 - Напоминания команде о дедлайнах
12:00 - Обновление статусов в Jira
14:00 - Grooming (снова записываем решения)
16:00 - Подготовка отчёта для руководства
```

> **Проблема:** 60% времени Scrum Master — это рутина, а не работа с командой

# СЛАЙД 3: ФУНКЦИОНАЛ

## 4 ключевые функции

### 1. Автосоздание задач из ТЗ
```
Вход: "Нужно сделать авторизацию через SMS"

AI создаёт в Jira:
├── Epic: "Авторизация через SMS"
├── Story: "Как пользователь, я хочу входить по SMS"
├── Task: "Backend: API отправки SMS"
├── Task: "Backend: Верификация кода"
├── Task: "Frontend: UI ввода номера"
├── Task: "Frontend: UI ввода кода"
└── Task: "QA: Тест-кейсы авторизации"
```

### 2. Транскрибация встреч
```
Встреча: "Решили использовать Twilio для SMS..."

AI фиксирует:
├── Решение: "SMS провайдер - Twilio"
├── Action Item: "Петров - интеграция Twilio до 15.02"
├── Blocker: "Нужен API ключ от финансов"
└── Обновляет задачу в Jira
```

### 3. Напоминания о дедлайнах
```
AI мониторит Jira и отправляет:
├── За 2 дня: "Задача X дедлайн послезавтра"
├── В день: "Сегодня дедлайн по задаче Y"
└── Просрочено: "Задача Z просрочена на 1 день"
```

### 4. Kanban доска с drag-and-drop
```
┌─────────┬─────────┬─────────┬─────────┐
│ Backlog │ To Do   │ In Prog │  Done   │
├─────────┼─────────┼─────────┼─────────┤
│ Task 1  │ Task 4  │ Task 6  │ Task 8  │
│ Task 2  │ Task 5  │ Task 7  │ Task 9  │
│ Task 3  │         │         │         │
└─────────┴─────────┴─────────┴─────────┘
```

# СЛАЙД 5: ТОЧНОСТЬ ФИКСАЦИИ (10 баллов)

## ≥90% решений зафиксированы после встреч

### Процесс транскрибации:

```
┌─────────────────────────────────────────┐
│         MEETING TRANSCRIPTION           │
├─────────────────────────────────────────┤
│                                         │
│  1. Запись встречи (Google Meet/Teams)  │
│     ↓                                   │
│  2. Speech-to-Text (Whisper API)        │
│     ↓                                   │
│  3. GPT анализ транскрипта              │
│     ├── Выделение решений               │
│     ├── Выделение action items          │
│     ├── Выделение blockers              │
│     └── Определение ответственных       │
│     ↓                                   │
│  4. Создание/обновление задач в Jira    │
│     ↓                                   │
│  5. Уведомление участников              │
│                                         │
└─────────────────────────────────────────┘
```

### Пример вывода:
```markdown
## Итоги встречи: Daily Stand-up 25.01.2025

### Решения:
1. Переносим релиз на 28.01 (согласовано с PM)
2. Используем Redis для кеширования

### Action Items:
| Задача | Ответственный | Дедлайн |
|--------|---------------|---------|
| Настроить Redis | Иванов | 26.01 |
| Обновить документацию | Петров | 27.01 |

### Blockers:
- Ждём API ключи от партнёра (эскалация на PM)
```
